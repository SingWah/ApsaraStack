state_machine service/state_machine.py

核心状态机引擎，继承 BaseEngine。使用 PriorityQueue 按优先级调度任务，4 个工作线程并行消费。每个任务设置 Timer 超时（默认 300s），超时后自动标记失败并触发重试。状态转换通过预定义的状态转移表驱动，支持回调钩子。
  • StateMachine extends BaseEngine
  • transition(state, event)
  • poll_status()
  • _worker_loop(queue)

常见异常与陷阱

问题影响建议
Timer 超时后未清理线程资源 长时间运行后线程泄漏 在 Timer callback 中显式 join 并释放
PriorityQueue 无界增长 突发流量时内存 OOM 设置 maxsize 并在满时丢弃低优先级任务
HTTP 调用无重试机制 网络抖动导致状态不一致 增加指数退避重试（最多 3 次）
